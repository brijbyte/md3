## Install

```bash
pnpm add @brijbyte/md3-react
```

The library is a styled layer over [Base UI](https://base-ui.com): every component wraps a
headless Base UI primitive, so behavior, accessibility, and keyboard handling come from
Base UI while the visuals follow the [Material Design 3](https://m3.material.io/) spec.

## Load the styles

JavaScript and CSS are decoupled — importing a component does **not** pull in its styles.
Import the stylesheet once at your app root:

```tsx
import "@brijbyte/md3-react/styles.css";
```

Or, for a minimal payload, import the shared layers plus only the components you use:

```tsx
import "@brijbyte/md3-react/tokens.css"; // required: design tokens + theming
import "@brijbyte/md3-react/ripple.css"; // required by pressable components
import "@brijbyte/md3-react/button.css";
```

The typescale expects Roboto; load it however you prefer, for example:

```html
<link
  href="https://fonts.googleapis.com/css2?family=Roboto:wght@400;500;700&display=swap"
  rel="stylesheet"
/>
```

## Use a component

There is no barrel export — import each component from its own path for better
tree-shaking and code-splitting:

```tsx
import { Button } from "@brijbyte/md3-react/button";
import AddIcon from "@brijbyte/md3-icons/outlined/add";

<Button variant="tonal" icon={<AddIcon />}>
  Add item
</Button>;
```

<Demo>
  <Button icon={<AddIcon />}>Add item</Button>
  <Button variant="tonal" icon={<AddIcon />}>

  </Button>
  <Button variant="outlined">Outlined</Button>
</Demo>

Props pass through to the underlying Base UI primitive, so everything it supports —
`render`, `disabled`, event handlers — works unchanged:

<Demo>
  <Checkbox defaultChecked aria-label="Demo checkbox" />
  <Switch defaultChecked aria-label="Demo switch" />
  <Button disabled>Disabled</Button>
</Demo>

## Theming

Colors are CSS custom properties (`--md-sys-color-*`). Light is the default; dark kicks in
via `prefers-color-scheme`, or explicitly with a `data-theme` attribute on `<html>`:

```html
<html data-theme="dark"></html>
```

Try it with the theme toggle in the header — every component on this page follows.

## Server components

Components are marked `'use client'`, so they can be imported and rendered directly from
React Server Component trees (this page is itself a server-rendered MDX file with live
client components inside it).

## Tailwind CSS

All library CSS lives in the `md3.tokens` / `md3.components` cascade layers, so unlayered
consumer CSS — including Tailwind utilities — always wins overrides. See
[Tailwind overrides](/tailwind) for the full setup, including using MD3 tokens as
Tailwind utilities (`bg-primary`, `text-title-large`, `shadow-level1`, …).
