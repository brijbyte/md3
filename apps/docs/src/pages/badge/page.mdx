## Variants

A small badge is a 6dp dot; give it children and it becomes the large badge
(16dp, label-small). Overflow formatting like "999+" is up to you. Badges also attach to tab items
— see [Tabs](/components/tabs).

## Accessibility

Per the MD3 spec, a badge's accessible label reads _after_ the destination it decorates, so
nest it inside the labelled control (icon button, tab, nav item) rather than placing it
elsewhere in the DOM. A numbered badge needs no extra label — its visible text ("5", "999+")
is what's announced. A dot badge has no visible text, so it defaults `aria-label` to "New
notification"; pass your own `aria-label` (or `aria-labelledby`) to override it. Stick to the
default token colors — they're already tuned for the spec's 3:1 container/label contrast
minimum, which a custom color pairing could break. Hiding the badge once its notification is
addressed is an app-level concern (not something the component can infer).
