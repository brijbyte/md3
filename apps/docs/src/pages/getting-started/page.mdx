## Install

```sh
npm install @brijbyte/md3-react
# or
pnpm add @brijbyte/md3-react
```

The library is a styled layer over [Base UI](https://base-ui.com): majority of the component
wraps a headless Base UI primitive, so behavior, accessibility, and keyboard handling come from
Base UI while the visuals follow the [Material Design 3](https://m3.material.io/) spec.

## Load the styles

JavaScript and CSS are decoupled — importing a component does **not** pull in its styles.
Import the shared layers plus only the components you use, so the CSS payload stays
minimal. Add these imports to your entry-point css file (e.g. `index.css`):

```css
/* required: tokens + theming */
@import "@brijbyte/md3-react/tokens.css";
/* required by most pressable components */
@import "@brijbyte/md3-react/ripple.css";
```

and then import the component css you need wherever required:

```css
@import "@brijbyte/md3-react/button.css";
```

> [!NOTE]
> There's also an everything bundle — `@import "@brijbyte/md3-react/styles.css"` — but it
> ships every component's CSS whether you use it or not; prefer the per-component imports.

The typescale expects Roboto; load it however you prefer, for example:

```html
<link
  href="https://fonts.googleapis.com/css2?family=Roboto:wght@400;500;700&display=swap"
  rel="stylesheet"
/>
```

## Use a component

Import each component (and its css) from its own path:

```tsx
import { Button } from "@brijbyte/md3-react/button";
import AddIcon from "@brijbyte/md3-icons/outlined/Add";

import "@brijbyte/md3-react/button.css";

<Button variant="tonal" icon={<AddIcon />}>
  Add item
</Button>;
```

## Theming

Colors are CSS custom properties (`--md-sys-color-*`). Light is the default; dark kicks in
via `prefers-color-scheme`, or explicitly with a `data-theme` attribute on `<html>`:

```html
<html data-theme="dark"></html>
```

## Tailwind CSS

All library CSS lives in the `theme` / `components` cascade layers (Tailwind v4's own
layer names), so unlayered consumer CSS — including Tailwind utilities — always wins
overrides. See
[Tailwind overrides](/overview/integration) for the full setup, including using MD3 tokens as
Tailwind utilities (`bg-primary`, `text-title-large`, `shadow-level1`, …).
